## Краткое описание системы

Веб-приложение на основе Spring Boot для управления пользователями с:

- **Аутентификацией** по email и паролю (Spring Security, BCrypt)
- **Ролевой моделью**: ROLE_ADMIN (полный CRUD) и ROLE_USER (только просмотр)
- **REST API** для управления пользователями (GET, POST, PATCH, DELETE)
- **Веб-интерфейсом** на Thymeleaf + Bootstrap + JavaScript (Fetch API)
- **Базой данных** PostgreSQL 16 с таблицами users, roles, user_role в схеме приложения `admin_panel`

## Технологический стек

- Java 21, Spring Boot 3.3.7
- Spring Security, Spring Data JPA, Thymeleaf
- PostgreSQL 16, Liquibase как отдельный контейнер, Hibernate (validate mode)
- Docker Compose
- Bootstrap 4/5, Vanilla JS (Fetch API)
- ModelMapper 3.1.1
- Maven

## Локальная инфраструктура

Перед началом работы необходимо запустить PostgreSQL 16 и отдельный контейнер Liquibase. Compose-файл находится в каталоге `infra`, поэтому сначала нужно перейти в этот каталог:

```bash
cd infra
docker compose up -d
```

После запуска контейнеров база данных полностью готова к работе. Docker Compose автоматически создаёт пользователя PostgreSQL, базу данных `ai_course`, отдельную схему приложения `admin_panel`, запускает контейнер Liquibase, применяет все миграции и заполняет базу тестовыми данными. Никаких SQL-скриптов вручную выполнять не требуется.

Проверить состояние инфраструктуры можно из того же каталога `infra`:

```bash
docker compose ps
docker compose logs liquibase
```

Контейнер PostgreSQL должен остаться в состоянии `running`/`healthy`, а контейнер Liquibase после успешного применения миграций завершится с кодом `0`. Для остановки инфраструктуры нужно находиться в каталоге `infra` и выполнить:

```bash
docker compose down
```

Студент не должен создавать пользователя, базу данных, схему, выполнять SQL-скрипты или импортировать данные вручную. Java-приложение должно явно использовать схему `admin_panel`; схема `public` не используется.

Liquibase не используется внутри Java-проекта: приложение не подключает Liquibase, не хранит файлы миграций в ресурсах и не запускает миграции средствами Spring Boot.
