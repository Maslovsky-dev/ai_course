# 11. Валидация и обработка ошибок

## 11.1. Валидация

### 11.1.1. Bean Validation (на уровне DTO)

Валидация выполняется на уровне UserDTO с использованием аннотаций JSR-380 (Bean Validation).

| Поле | Аннотация | Сообщение об ошибке | Условие |
|---|---|---|---|
| `firstName` | `@NotEmpty` | "Name should be not empty" | Не должно быть пустым |
| `firstName` | `@Size(min=2, max=30)` | "Name wrong size" | Длина от 2 до 30 символов |
| `lastName` | `@NotEmpty` | "LastName should be not empty" | Не должно быть пустым |
| `lastName` | `@Size(min=2, max=30)` | "Name wrong size" | Длина от 2 до 30 символов |
| `age` | `@Min(18)` | "Age > 18" | Минимум 18 |
| `age` | `@Max(150)` | "Age < 150" | Максимум 150 |
| `age` | `@NotNull` | "Age should be not empty" | Не должно быть null |
| `email` | `@NotEmpty` | "Email should be not empty" | Не должно быть пустым |
| `email` | `@Size(min=2, max=100)` | "Email size error" | Длина от 2 до 100 символов |
| `password` | `@NotEmpty` | "Password should be not empty" | Не должно быть пустым |
| `roles` | `@NotEmpty` | "Не выбрана ни одна роль" | Должна быть хотя бы одна роль |

### 11.1.2. Bean Validation (на уровне сущности User)

Сущность User также содержит аннотации валидации (используются при серверном рендеринге, но не при REST API):

| Поле | Аннотация | Сообщение об ошибке |
|---|---|---|
| `firstName` | `@NotEmpty` | "Имя не должно быть пустым" |
| `firstName` | `@Size(min=2, max=30)` | "Имя должно быть от 2 до 100 символов" |
| `lastName` | `@NotEmpty` | "Фамилия не должна быть пустым" |
| `lastName` | `@Size(min=2, max=30)` | "Фамилия должна быть от 2 до 100 символов" |
| `age` | `@Min(18)` | "Пользоваться сервисом можно только с 18 лет" |
| `age` | `@Max(150)` | "Возраст превышает допустимый" |
| `age` | `@NotNull` | "Возраст не должен быть пустым" |
| `email` | `@NotEmpty` | "Почта не должна быть пустой" |
| `email` | `@Size(min=2, max=100)` | "Почта должно быть от 2 до 100 символов" |
| `password` | `@NotEmpty` | "Пароль не должен быть пустым" |
| `roles` | `@NotEmpty` | "Не выбрана ни одна роль" |

### 11.1.3. Кастомная валидация (UserValidator)

Дополнительная валидация уникальности email при **создании** пользователя.

| Проверка | Условие | Сообщение об ошибке |
|---|---|---|
| Уникальность email | Если в БД существует пользователь с таким же email | "Человек с такой почтой уже существует" (поле: email) |

**Алгоритм UserValidator**:

1. Принимает объект User и Errors.
2. Вызывает `UserDetailsServiceImpl.loadUserByUsername(user.getEmail())`.
3. Если пользователь найден (исключение не выброшено) — добавляет ошибку в поле "email".
4. Если пользователь не найден (выброшено `UsernameNotFoundException`) — валидация прерывается без ошибки (email уникален).

> **Важно**: Данная валидация применяется **только при создании** пользователя (POST). При обновлении (PATCH) эта проверка не выполняется.

### 11.1.4. Когда применяется валидация

| Операция | Bean Validation | UserValidator (уникальность email) |
|---|---|---|
| Создание (POST `/users/`) | ✅ | ✅ |
| Обновление (PATCH `/users/{id}`) | ✅ | ❌ |
| Удаление (DELETE `/users/{id}`) | ❌ | ❌ |
| Получение (GET) | ❌ | ❌ |

## 11.2. Обработка ошибок

### 11.2.1. Исключения

| Исключение | Тип | Когда выбрасывается | HTTP-код | Сообщение |
|---|---|---|---|---|
| `UserNotFoundException` | RuntimeException | При запросе пользователя по несуществующему ID (GET) | 404 | "User with this id wasn't found" |
| `UserNotCreatedException` | RuntimeException | При ошибках валидации при создании/обновлении (POST/PATCH) | 400 | Строка с описанием всех ошибок валидации |
| `UsernameNotFoundException` | Spring Security | При аутентификации с несуществующим email | — (редирект на login?error) | "User not found!" |

### 11.2.2. Обработчики исключений (Exception Handlers)

Обработчики определены в REST-контроллере.

#### Обработчик UserNotFoundException

```json
{
  "message": "User with this id wasn't found",
  "timestamp": 1700000000000
}
```

- HTTP-статус: 404 NOT FOUND
- Тело: UserErrorResponse с фиксированным сообщением и текущим timestamp.

#### Обработчик UserNotCreatedException

```json
{
  "message": "email - Человек с такой почтой уже существует; age - Age > 18",
  "timestamp": 1700000000000
}
```

- HTTP-статус: 400 BAD REQUEST
- Тело: UserErrorResponse с сообщением из исключения и текущим timestamp.
- Сообщение формируется из всех ошибок валидации, объединённых через `;` в формате `поле - сообщение`.

### 11.2.3. Формирование сообщения об ошибках валидации

**Алгоритм**:

1. Из `BindingResult` получаются все `FieldError`.
2. Каждый error преобразуется в строку: `error.getField() + " - " + error.getDefaultMessage()`.
3. Все строки объединяются через `;`.
4. Полученная строка передаётся в `UserNotCreatedException`.

Пример: `"firstName - Name should be not empty; age - Age > 18; roles - Не выбрана ни одна роль"`

### 11.2.4. Обработка ошибок на клиенте (JavaScript)

| Сценарий | Обработка |
|---|---|
| Ошибка при `getAllUsers()` | `console.error(error)` |
| Ошибка при `getUser()` | `console.error(error)` |
| Ошибка при создании (newUser) | `console.error('Error creating user:', error)` |
| Ошибка при обновлении (response not ok) | `console.error('Error updating user:', response)` + `alert('Error updating user')` |
| Ошибка при обновлении (network error) | `console.error('Error updating user:', error)` + `alert('Error updating user')` |
| Ошибка при удалении (response not ok) | `console.error('Error updating user:', response)` + `alert('Error updating user')` |
| Ошибка при удалении (network error) | `console.error('Error updating user:', error)` + `alert('Error updating user')` |

### 11.2.5. Обработка ошибок аутентификации

| Сценарий | Обработка |
|---|---|
| Неверный email или пароль | Редирект на `/auth/login?error`, отображение "Неправильные имя или пароль" |
| Доступ к защищённому ресурсу без аутентификации | Редирект на `/auth/login` |
| Доступ к ресурсу ADMIN с ролью USER | HTTP 403 Forbidden |
