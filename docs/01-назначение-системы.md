# 1. Назначение системы

## 1.1. Общее описание

Разрабатываемая система представляет собой веб-приложение для управления пользователями с ролевой моделью доступа. Система предназначена для администрирования учётных записей пользователей, управления их ролями и просмотра информации о пользователях.

## 1.2. Основные цели

- Обеспечить аутентификацию и авторизацию пользователей на основе email и пароля.
- Предоставить административный интерфейс для полного управления учётными записями (создание, просмотр, редактирование, удаление).
- Предоставить пользовательский интерфейс для просмотра собственных данных.
- Обеспечить разграничение доступа на основе ролей: **ADMIN** и **USER**.

## 1.3. Область применения

Система является демонстрационным приложением управления пользователями и может быть использована как основа для модуля администрирования пользователей в более крупных веб-приложениях.

## 1.4. Пользователи системы

| Пользователь | Описание |
|---|---|
| **Администратор (ADMIN)** | Имеет доступ к панели администрирования, может управлять всеми пользователями системы: создавать, просматривать, редактировать и удалять учётные записи. Также может просматривать собственную информацию. |
| **Обычный пользователь (USER)** | Имеет доступ только к странице с информацией о себе. Не может управлять другими пользователями. |

## 1.5. Ключевые возможности

1. **Аутентификация** — вход в систему по email и паролю.
2. **Управление пользователями (CRUD)** — полный набор операций для администратора.
3. **Ролевая модель** — каждый пользователь имеет одну или несколько ролей.
4. **Защита паролей** — пароли хранятся в зашифрованном виде (BCrypt).
5. **Разграничение доступа** — различные страницы и API-эндпоинты доступны в зависимости от роли.
6. **REST API** — управление пользователями осуществляется через RESTful API.
7. **Веб-интерфейс** — пользовательский интерфейс на основе серверного рендеринга (Thymeleaf) и клиентского JavaScript (Fetch API).
