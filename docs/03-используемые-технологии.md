## 3.4. Инфраструктура

| Компонент | Описание |
|---|---|
| **Сервер приложений** | Встроенный Apache Tomcat (из spring-boot-starter-web) |
| **СУБД** | PostgreSQL 16, база данных `ai_course`, схема приложения `admin_panel` |
| **Миграции** | Отдельный контейнер Liquibase автоматически применяет миграции при запуске Docker Compose |
| **Локальная инфраструктура** | PostgreSQL 16 и отдельный контейнер Liquibase запускаются через Docker Compose |
| **Порт по умолчанию** | 8080 |
| **Сборка и запуск** | Maven (`mvn spring-boot:run`) или упаковка в JAR (`mvn package`) |

## 3.5. Требования к окружению

- **JDK**: версия 21
- **Maven**: для сборки проекта (либо используется обёртка `mvnw`)
- **Docker и Docker Compose**: для запуска локальной инфраструктуры
- **Браузер**: современный браузер с поддержкой Fetch API и ES6

Перед началом работы необходимо запустить локальную инфраструктуру:

- PostgreSQL 16
- отдельный контейнер Liquibase

Compose-файл находится в каталоге `infra`, поэтому запуск выполняется из этого каталога:

```bash
cd infra
docker compose up -d
```

После запуска контейнеров база данных полностью готова к работе. Docker Compose автоматически создаёт пользователя PostgreSQL, базу данных `ai_course`, отдельную схему приложения `admin_panel`, запускает контейнер Liquibase, применяет все миграции и заполняет базу тестовыми данными. Никаких SQL-скриптов вручную выполнять не требуется.

Проверка состояния выполняется из каталога `infra` командами `docker compose ps` и `docker compose logs liquibase`. PostgreSQL должен быть запущен и доступен на `localhost:5432`, контейнер Liquibase после успешного применения миграций должен завершиться с кодом `0`.

Liquibase не должен подключаться к Java-проекту и не должен запускаться средствами Spring Boot. Файлы миграций не хранятся внутри ресурсов Java-приложения.

Во всех примерах конфигурации Java-приложения необходимо явно использовать схему приложения `admin_panel`. Использование схемы `public` не допускается.
